# Guia de Leitura da Documentação

Este guia ajuda a navegar rapidamente pelos documentos, seja em leitores de Markdown ou via MKDocs.

---

## 1. Ordem sugerida de leitura
1. [wiki/ideias.md](wiki/ideias.md) — visão do produto e princípios.
2. [wiki/regras-de-negocio.md](wiki/regras-de-negocio.md) — comportamento obrigatório.
3. [wiki/modelos-json.md](wiki/modelos-json.md) — estrutura de dados e persistência.
4. [wiki/padroes-minimos.md](wiki/padroes-minimos.md) — padrões de organização do projeto.
5. [pendencias.md](pendencias.md) — dúvidas e decisões pendentes.

---

## 2. Onde registrar dúvidas
Se algo não estiver definido nos documentos de fonte de verdade:
- **Não implementar**.
- Registrar em [pendencias.md](pendencias.md) usando o formato obrigatório.

---

## 3. Convenções de leitura
- **Títulos curtos**: cada seção deve ter um propósito claro.
- **Listas objetivas**: regras e decisões ficam em bullets.
- **Links relativos**: funcionam no editor e no MKDocs.

---

## 4. Como navegar no MKDocs
Para abrir o site local:

```bash
mkdocs serve
```

Use o menu lateral para acessar wiki, regras e board.
